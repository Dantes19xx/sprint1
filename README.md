Проектирование структуры проекта "Mesto" (из монолита в микрофронтенды).

Уровень 1. Проектирование. Выбор фреймворка.
На мой взгляд целесообразнее в данном проекте использовать фреймворк WMF, т.к. Приложение написано с использованием только реакта, и не требует объединения разных микрофронтендов написанных с использованием разных фреймворков.

Уровень 2. Планирование изменений
Патерн декомпозиции тут стоит выбрать вертикальную нарезку и вынести все компоненты и стили, которые используют их формы в отдельностоящие микрофронтенды.

В данном монолите можно выделить следующие домены:
-Авторизация (auth);
-Работа с профилем пользователя. Создание/редактирование (profile);
-Загрузка и удаление фото (card);
-Сбор лайков под фото (like_audit).

Моя структура будет выглядть следующим образом:
/auth
    /src
        /components
            Login.js
            Register.js
        /styles
            тут лежат все стили, которые испольщуются компонентами Login и Register
    config.js 
    package.json
